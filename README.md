# Kanban Task Manager

## Описание

**java-kanban** — это учебный проект, реализующий систему управления задачами по методологии Kanban на языке Java. Проект предназначен для управления задачами, эпиками и подзадачами с отслеживанием их статусов и взаимодействий между ними.

## Основные возможности

- CRUD для задач, эпиков и подзадач
- Отслеживание истории просмотров задач
- Приоритизация задач и предотвращение пересечений по времени
- Хранение задач в памяти и сериализация данных

## Используемые технологии и зависимости

- **Язык разработки:** Java (версия 11 или выше)
- **Библиотеки и зависимости:**
    - **JUnit 5** — для написания и запуска модульных тестов.
    - **Gson** (или Jackson) — для сериализации и десериализации объектов в JSON (используется, если задачи и история сохраняются в файловом формате или обмениваются по сети).  
    - **HttpServer из JDK** — минималистичный встроенный сервер, реализующий REST API (используется для обработки HTTP-запросов к задачам).
    - **Lombok** *(по желанию)* — для сокращения шаблонного кода (геттеры, сеттеры, конструкторы).
- **Сборщик проекта:** Macen

## Структура проекта

- `model` — модели данных (Task, Epic, Subtask)
- `service` — бизнес-логика и менеджеры (TaskManager, HistoryManager)
- `httpHandler` — обработчики HTTP-запросов
- `tests` — модульные тесты

## API эндпоинты

Если в проекте реализован HTTP-сервер, доступны следующие REST эндпоинты:

### Эндпоинты для работы с задачами

| Метод   | URL                     | Описание                          |
|---------|-------------------------|-----------------------------------|
| GET     | `/tasks`                | Получить все задачи (Task, Epic, Subtask) в порядке приоритета |
| GET     | `/tasks/task`           | Получить список всех задач        |
| GET     | `/tasks/task/{id}`      | Получить задачу по ID             |
| POST    | `/tasks/task`           | Создать новую задачу или обновить существующую (по ID) |
| DELETE  | `/tasks/task/{id}`      | Удалить задачу по ID              |

### Эндпоинты для работы с эпиками

| Мет��д   | URL                     | Описание                          |
|---------|-------------------------|-----------------------------------|
| GET     | `/epics`           | Получить список всех эпиков       |
| GET     | `/epics/epic/{id}`      | Получить эпик по ID               |
| POST    | `/epics/epic`           | Создать новый эпик или обновить существующий (по ID) |
| DELETE  | `/epics/epic/{id}`      | Удалить эпик по ID (удаляются и связанные подзадачи) |

### Эндпоинты для работы с подзадачами

| Метод   | URL                       | Описание                        |
|---------|---------------------------|---------------------------------|
| GET     | `/subtasks`          | Получить список всех подзадач   |
| GET     | `/subtasks/subtask/{id}`     | Получить подзадачу по ID        |
| GET     | `/subtasks/subtask/epic/{epicId}` | Получить все подзадачи определенного эпика |
| POST    | `/subtasks/subtask`          | Создать новую подзадачу или обновить существующую (по ID) |
| DELETE  | `/subtasks/subtask/{id}`     | Удалить подзадачу по ID         |

### Прочие эндпоинты

| Метод   | URL               | Описание                       |
|---------|-------------------|-------------------------------|
| GET     | `/history`  | Получить историю просмотров задач |
| GET     | `/prioritized`  | Сортировка задач по приоритету (времени начала) |


## Как запустить проект

1. Склонируйте репозиторий:
    ```sh
    git clone https://github.com/ErmolaevSD/java-kanban.git
    ```
2. Откройте проект в вашей IDE.
3. Соберите проект через Maven или Gradle.
4. Запустите основной класс приложения (`Main`) или серверный класс (`HttpTaskServer`, если реализован).
